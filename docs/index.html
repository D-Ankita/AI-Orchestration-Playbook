2 Schema-Guided Generation

The AI can only use components you've registered. No hallucinated types, no invented props.

// Register what the AI is allowed to generate
registry.register({
  type: 'Card',
  props: {
    title: { type: 'string', required: true },
    variant: { type: 'enum', enum: ['default', 'elevated'] },
  },
  slots: {
    footer: { allowedTypes: ['Button'], maxChildren: 3 }
  }
});

// AI tries to generate <MagicWidget> → rejected
// AI generates <Card title="Pricing"> → accepted

Complex tasks need multiple LLM turns. The agent inspects state, looks up schemas, and validates work before responding.

5 Zod-Validated Tools

Every tool validates input AND output with Zod at runtime. Built-in timeout, retry with backoff, and abort.

const tool = defineTool({
  name: 'fetch_cms_data',
  inputSchema: z.object({
    contentType: z.string(),
    limit: z.number().default(10),
  }),
  outputSchema: z.object({
    entries: z.array(z.record(z.unknown())),
  }),
  timeout: 10_000,
  retryPolicy: { maxRetries: 2, backoffMs: 1000 },
});

// AI sends {contentType: 42} → Zod rejects at boundary
// Tool returns {invalid: true} → Zod rejects output
// Tool hangs → times out after 10s → retries twice

6 4-Layer Validation

AI output passes through four independent safety layers before touching the application.

LAYER 1
Schema Conformance Valid component types? Props exist? Slots defined?
<MagicWidget> → rejected
LAYER 2
Structural Integrity Unique UIDs? Parents exist? No circular refs?
duplicate uid → rejected
LAYER 3
Safety Guardrails No scripts, event handlers, iframes, external resources
<script> → blocked
LAYER 4
Checksum Verification SHA-256 checksum confirms state hasn't drifted since generation started
stale edit → warned

7 Streaming with Backpressure

Real-time feedback as the AI generates. Typed events, buffer management, abort support.

// 7 typed event types flow through the stream:

text         → incremental AI response text
thinking     → model reasoning (when enabled)
tool_start   → AI wants to call a tool
tool_result  → tool execution completed
change       → a validated NodeChange to apply
done         → generation complete + token usage
error        → recoverable or fatal error

// Backpressure: if buffer > 1MB, stream pauses
// Abort: AbortController cancels mid-generation
